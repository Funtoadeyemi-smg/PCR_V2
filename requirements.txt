1. Navigate to the AWS Lambda console

2. Locate and select the Lambda function named 'pcr-deployment-function'

3. In the function details page, select the 'Monitor' tab

4. Check if there's a 'View logs in CloudWatch' button. If not, proceed to the next step

5. Go to the AWS CloudWatch console

6. In the navigation pane, select 'Log groups'

7. In the search bar, enter '/aws/lambda/pcr-deployment-function'

8. If the log group is not found, create a new log group:
   - Click on 'Create log group'
   - Enter '/aws/lambda/pcr-deployment-function' as the log group name
   - Click 'Create'

9. Return to the Lambda function console

10. Select the 'Configuration' tab

11. Click on 'Permissions' in the left sidebar

12. Scroll down to the 'Execution role' section

13. Click on the role name to open the IAM console in a new tab

14. In the IAM console, select 'Add permissions', then 'Create inline policy'

15. In the JSON tab, paste the following policy:
    ```json
    {
      "Version": "2012-10-17",
      "Statement": [
        {
          "Effect": "Allow",
          "Action": [
            "logs:CreateLogGroup",
            "logs:CreateLogStream",
            "logs:PutLogEvents"
          ],
          "Resource": "arn:aws:logs:*:619071338733:log-group:/aws/lambda/pcr-deployment-function:*"
        }
      ]
    }
    ```

16. Click 'Review policy'

17. Name the policy 'LambdaLogGroupAccess' and click 'Create policy'

18. Return to the Lambda function console and invoke the function to generate logs

19. Check the CloudWatch logs to ensure they are being created correctly
